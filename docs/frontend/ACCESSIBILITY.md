# Accessibility

## Purpose

This document defines accessibility standards for the Skillar Mobile application.

## Principles

- Accessibility by default.
- Inclusive user experience.
- Keyboard and screen reader support where applicable.
- Sufficient color contrast.
- Consistent navigation.

## Guidelines

- Provide accessibility labels for interactive elements.
- Ensure touch targets are appropriately sized.
- Do not rely solely on color to convey meaning.
- Support dynamic text sizing where practical.
- Maintain logical focus order.

## Testing

Accessibility should be validated during development and QA using platform accessibility tools and manual testing.

## Future Enhancements

This guide will be expanded with WCAG references, accessibility checklists, and component-specific recommendations.
