# Form Standards

## Purpose

This document defines standards for building forms in the Skillar Mobile application.

## Principles

- Keep forms simple and accessible.
- Validate input on both client and server.
- Provide clear error messages.
- Minimize user effort.

## Validation

- Validate required fields.
- Validate formats and ranges.
- Show inline validation where appropriate.
- Prevent duplicate submissions.

## UX Guidelines

- Use descriptive labels.
- Display loading indicators during submission.
- Preserve user input on validation errors.
- Support keyboard navigation where applicable.

## Future Enhancements

This guide will be expanded with validation patterns, reusable form components, and integration examples with the project's chosen form library.
