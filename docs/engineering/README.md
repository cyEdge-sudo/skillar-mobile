# Engineering Documentation

## Overview

This section defines the engineering standards, practices, governance, and reference material for the Skillar Mobile application.

The goal is to ensure that every contributor—human or AI—builds software consistently, predictably, securely, and maintainably.

---

### Principles

Defines the core engineering philosophy and architectural rules.

- Coding Standards
- Clean Architecture
- SOLID
- AI Development Guidelines

### Practices

Defines implementation guidance for TypeScript, React, Expo, testing, security, performance, build & release, and dependency management.

### Governance

Defines engineering quality gates including Definition of Ready, Definition of Done, review checklists, technical debt management, and engineering metrics.

### Reference

Contains reusable examples, anti-patterns, refactoring guidance, and architectural fitness rules.

## Engineering Philosophy

Skillar Mobile follows:

- Clean Architecture
- SOLID Principles
- DRY
- KISS
- YAGNI
- Composition over inheritance
- Convention over configuration
- Security by default
- Accessibility by default
- Documentation-first development

Documentation is considered a first-class artifact and must evolve alongside the codebase.
