# Skillar Mobile Documentation

Welcome to the official documentation for the Skillar Mobile application. This documentation is the authoritative source for the application's architecture, engineering standards, and implementation guidelines.

## Guiding Principles

- Documentation-first development
- Clean Architecture
- Feature-first modular design
- Secure-by-default implementation
- Offline-first user experience
- Accessibility by design
- Reusable design system
- Testability and maintainability

## Documentation Lifecycle

1. Define requirements.
2. Review and approve architecture.
3. Record significant architectural decisions using ADRs.
4. Implement features.
5. Update documentation alongside code changes.

## Repository Philosophy

The documentation is considered the project's source of truth. Significant implementation changes should be reflected in the relevant documentation before or alongside the code changes to minimize architectural drift.
